# 빠른 해결책: 현재 SQLite 유지 (임시)

## 현재 상황
- 데이터가 불러와지고 있음
- 하지만 재시작 시 초기화될 수 있음

## 임시 해결책

### 방법 1: 데이터베이스 파일을 외부 저장소에 백업

**장점:**
- 빠르게 구현 가능
- 코드 변경 최소화

**단점:**
- 완벽하지 않음
- 백업/복원 로직 필요

### 방법 2: 정기적으로 데이터베이스 덤프

**구현:**
1. 주기적으로 데이터베이스 백업
2. GitHub에 커밋
3. 시작 시 복원

### 방법 3: Vercel에서 데이터베이스 파일을 읽기 전용으로 사용

**현재 코드는 이미 이 방식:**
- 프로젝트 루트의 `data.db`를 `/tmp`로 복사
- 읽기는 가능하지만 쓰기는 제한적

## 권장: Supabase로 전환

**왜 Supabase?**
- ✅ 무료
- ✅ 영구 저장
- ✅ 자동 백업
- ✅ 서버리스 환경에 최적화
- ✅ Vercel과 잘 맞음

## 다음 단계

1. **지금은 현재 상태 유지**
   - 데이터가 불러와지고 있음
   - 재시작 시 문제가 발생하면 대응

2. **Supabase 설정 시작**
   - 무료 계정 생성
   - 데이터베이스 연결
   - 코드 전환 (점진적으로)

3. **데이터 마이그레이션**
   - SQLite → PostgreSQL
   - 데이터 이전

## 즉시 할 수 있는 것

현재 코드는 프로젝트 루트의 `data.db`를 `/tmp`로 복사하므로:
- ✅ 읽기는 가능
- ⚠️ 쓰기는 제한적 (재시작 시 사라질 수 있음)

**데이터 추가/수정이 필요하면:**
